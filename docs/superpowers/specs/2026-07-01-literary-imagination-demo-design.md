# 문학 상상력 증폭 데모 — 설계 문서 (Design Spec)

- **작성일**: 2026-07-01
- **상태**: 승인됨 (구현 계획 단계로 진행)
- **한 줄 요약**: ebook 텍스트의 여백을 시각·청각·촉각(유사)으로 **증폭**해, 어느 정도 글을 읽는 아이가 "이렇게 생생하게 상상할 수 있구나"를 체험하는 웹 데모.

---

## 1. 목표와 범위

### 목표
- 문학 텍스트를 **오감 체험**으로 바꿔 아이의 상상력을 자극한다.
- 4가지 감각 포인트: **Literary(원문 텍스트) · Visual(영상) · Auditory(사운드) · Tactile(진동감)**.
- 진동은 앱에서 직접 줄 수 없으므로 **화면 흔들기 + 진동 사운드**로 진동감을 흉내 낸다.

### 핵심 원리 — 감각적 증폭 (Sensory Amplification)
> 책: *"문이 닫혔다."*
> 체험: **쾅!**(문 소리) + 화면 흔들림 + 낮은 진동음 + 증폭 자막 *"집 전체가 울렸다"*

원문의 여백을 감각으로 채워, 상상을 **대체**하지 않고 **유도**한다. 데모에서는 생생한 증폭 쪽으로 가되, 균형은 조절 가능하게 둔다.

### 범위 (데모)
- 형태: **웹 페이지** (HTML + CSS + JavaScript + 가벼운 백엔드).
- **2단(tier) 구조**:
  - Tier 1 — 미리 제작한 **샘플 1개** (풀 AI 애니메이션, 최고 퀄리티).
  - Tier 2 — 사용자가 **책 사진 업로드 → 실시간 자동 생성** 체험.
- 초기 UI는 **선택지 1개**로 시작(샘플 체험). 업로드 진입도 제공.

---

## 2. 사용자 흐름

```
[시작 화면]
 ├─ ① 샘플 체험   → 미리 만든 풀 애니메이션 감각 체험 재생
 └─ ② 내 책으로 하기 → 사진 업로드 → 산출물 순차 공개 → 최종 완성본
```

### Tier 1 — 샘플 (미리 제작)
- 레퍼런스 그림체 → 장면 일러스트 → **image-to-video 클립**(미리 생성).
- 원문 + 코드 감각 효과(흔들기·번쩍·사운드·증폭 자막·내레이션)를 **손으로 정교하게** 타임라인에 배치.
- "이런 게 가능하다"는 임팩트 담당.

### Tier 2 — 사용자 업로드 (실시간, 산출물 스트리밍 공개)
파이프라인 각 단계 결과가 **나오는 즉시** 화면에 공개되고, 다음 단계가 준비되면 부드럽게 교체된다. 빈 로딩 화면이 없다.

```
사진 업로드
 → "글자를 읽는 중..."      → Claude가 원문 텍스트 추출 → 타이핑되듯 등장 (스트리밍)  [초]
 → "장면을 그리는 중..."     → 그림체 일러스트 생성 → 표시                          [초]
 → "움직임을 불어넣는 중..."  → Veo 영상 완성 → 일러스트를 영상으로 교체              [수십초~분]
 → "감각을 입히는 중..."     → 흔들기·소리·증폭 자막·내레이션 얹혀 최종 완성
```

- 기다림 자체가 **"상상이 조립되는 쇼"** = 데모의 하이라이트.
- 구현은 표준 패턴(백엔드 파이프라인 + SSE/폴링으로 부분 산출물 스트리밍).

---

## 3. 콘텐츠 모델 — 비트(beat) 타임라인 ⭐ (심장)

장면 텍스트를 여러 **순간(beat)**으로 분해하고, 각 비트에 원문 + 증폭 연출을 붙인다.

```
beat = {
  text:            "문이 닫혔다.",          // 원문
  amplifiedCaption:"집 전체가 울렸다",       // 상상 자극용 확장 묘사
  clip:            "door_close.mp4",         // (Tier1) 영상 / (Tier2) 생성물
  image:           "door_closed.png",        // (Tier2 프리뷰) 정지 이미지
  effects: [
    { type: "shake", intensity: "high", duration: 600 },
    { type: "flash", color: "#000", strength: 0.2 },
    { type: "sound", src: "door_slam.mp3" },
    { type: "sound", src: "low_rumble.mp3" }   // 진동음(우웅)
  ],
  narration:       "문이 닫혔다."             // (선택) 읽어주기 TTS
}
```

- 이 **비트 = 원문 + 감각 증폭** 구조가 프로젝트의 심장.
- 차이는 "비트를 사람이 짰나(Tier 1) vs Claude가 짰나(Tier 2)"뿐. **타임라인 엔진과 효과 부품은 공유**.

---

## 4. 공통 엔진 & 감각 효과 부품

두 tier가 공유하는 재사용 부품:

| 부품 | 역할 |
|------|------|
| `playClip(src)` | 영상 클립 재생 (배경 레이어) |
| `showImage(src)` | 정지 일러스트 표시 (Tier 2 프리뷰) |
| `shake(intensity, duration)` | 카메라 흔들림 (요소 transform 지터) |
| `flash(color, strength)` | 번쩍임 오버레이 (번개·충격) |
| `playSound(src)` | 효과음/배경음/진동음 |
| `showText(text, mode)` | 원문 / 증폭 자막 등장 |
| `narrate(text|audio)` | (선택) 읽어주기 내레이션 |

**타임라인 엔진**: 비트를 순서대로 읽어 시간축에 맞춰 위 부품들을 트리거한다.

## 5. 기술 구성

### 프런트엔드
- 순수 **HTML + CSS + JavaScript** (무거운 프레임워크 없이 시작, 필요 시 확장).
- 영상 레이어: `<video>`.
- 효과 레이어: CSS transform/animation (흔들기·확대·페이드), 오버레이 div(번쩍임).
- 사운드: Web Audio API / `<audio>`.

### 백엔드 (가벼운 서버)
- 목적: AI API 키 보호 + Tier 2 파이프라인 오케스트레이션.
- 업로드 이미지를 받아 각 AI 서비스를 순차 호출, **부분 산출물을 SSE/폴링으로 프런트에 스트리밍**.
- Claude 응답은 토큰 스트리밍을 그대로 프런트로 전달("책을 읽어나가는" 연출).

### AI 서비스 스택 (비용 무관, 최고 티어)
| 파이프라인 역할 | 도구 |
|----------------|------|
| **두뇌**(이미지 읽기 + 비트 분해 + 증폭 자막 + 효과 설계) | **Claude Opus 4.8** (vision) |
| **그림체 일러스트 생성** | **Nano Banana (Gemini 2.5 Flash Image)** 또는 Midjourney v7 |
| **image-to-video (애니메이션)** | **Google Veo 3** (영상+오디오 동시 생성) / Kling 2.x / Runway Gen-4 |
| **효과음(foley)** | **ElevenLabs Sound Effects** |
| **배경 음악** | **Suno v4** |
| **읽어주기 내레이션 (선택 포함)** | **ElevenLabs TTS** |

> 주의: AI 도구 최신 버전/가용성은 빠르게 변하므로 구현 착수 시 재확인한다. SDK/API 사용 전 공식 문서 확인.

### OCR 관련
- Tier 1(샘플): OCR은 **개발 단계에서만** 사용(사진→텍스트). 데모 실행 시엔 사용 안 함.
- Tier 2(업로드): 별도 OCR 없이 **Claude vision이 이미지에서 직접 텍스트 추출 + 비트 설계**를 한 번의 호출로 처리.

---

## 6. 준비물 (사용자 제공)
- 샘플용 **책 사진 또는 텍스트**.
- **레퍼런스 이미지** (그림체 기준).
- (착수 후 함께) 이미지 생성 → 클립 생성 도구 접근, 각 AI 서비스 **API 키**.

---

## 7. 미해결/착수 시 결정할 사항
- 샘플 장면 텍스트/사진 및 레퍼런스 이미지 확보.
- Tier 2 실시간 파이프라인의 실패/타임아웃 처리(예: Veo 지연 시 코드 효과 버전으로 폴백할지).
- 진동음/효과음 사운드 에셋 목록.
- 내레이션 on/off 토글 UX.

---

## 8. 설계 원칙
- **YAGNI**: 데모에 불필요한 기능은 넣지 않는다(초기엔 샘플 1개 + 업로드 1경로).
- **부품화**: 감각 효과는 재사용 부품으로 만들어 새 책도 쉽게 추가.
- **관심사 분리**: 비트 데이터(콘텐츠) ↔ 타임라인 엔진(재생) ↔ 효과 부품(연출)을 분리.
